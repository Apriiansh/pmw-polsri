# Kisah: Fleksibilitas Draft Proposal & Standarisasi Anggota

**Waktu**: 17 April 2026, 07:10 PM
**Pelaku**: Antigravity (AI Architect)

## Apa yang Terjadi?

Kami melakukan pembaruan pada modul **Proposal Mahasiswa** untuk memberikan fleksibilitas saat pengisian draft dan menyelaraskan aturan jumlah anggota tim.

### 1. Fleksibilitas Simpan Draft
Mahasiswa kini dapat menyimpan draft proposal meskipun data belum terisi lengkap (seperti nama usaha, dosen, atau anggota). 
- **Frontend**: Menambahkan atribut `formnovalidate` pada tombol "Simpan Draft" untuk melewati validasi browser.
- **Backend**: Mengubah logika validasi di `ProposalController`. Validasi wajib (`required`) hanya ditegakkan jika mahasiswa melakukan **Final Submit**.

### 2. Standarisasi Jumlah Anggota
Mengikuti permintaan user untuk menetapkan jumlah tim menjadi **4-5 orang**, kami memperbarui batasan anggota:
- **Aturan Baru**: 1 Ketua + 3 sampai 4 Anggota (Total 4-5 orang).
- **Update Kode**: 
    - Controller kini menolak final submit jika anggota kurang dari 3 atau lebih dari 4.
    - Alpine.js `init()` pada form baru secara otomatis menambahkan 3 baris anggota awal.
    - Semua label instruksi di UI diperbarui menjadi "Wajib 3-4 anggota (Total tim 4-5 orang)".

### 3. Peningkatan UX
- Tombol **"Kirim Proposal Sekarang"** hanya akan aktif (enabled) jika:
    - Data utama (Dosen, Kategori, Nama Usaha, Kategori Wirausaha) sudah terisi.
    - Jumlah anggota sudah memenuhi syarat (3-4 orang).
    - Seluruh dokumen wajib sudah diunggah atau dipilih.
