# Kisah: Penghapusan index.php dari URL

## Masalah
URL pada saat deploy masih menyertakan `index.php` (contoh: `https://simpmw.polsri.ac.id/index.php/dashboard`). Hal ini membuat URL terlihat kurang profesional dan tidak mengikuti standar Clean URL.

## Langkah Perbaikan
1.  **Konfigurasi App.php**:
    - Mengubah `public string $indexPage = 'index.php';` menjadi `public string $indexPage = '';`. Ini adalah langkah utama agar helper URL CodeIgniter tidak menyertakan file index tersebut.
    - Mengupdate `public string $baseURL = 'https://simpmw.polsri.ac.id/';` sebagai nilai default produksi.
2.  **Verifikasi .htaccess**:
    - Memastikan file `public/.htaccess` sudah memiliki aturan `mod_rewrite` yang benar untuk mengarahkan request ke `index.php` secara internal. File tersebut sudah terkonfigurasi dengan benar.

## Status Saat Ini
- Konfigurasi aplikasi sudah mendukung Clean URL.
- URL sekarang dapat diakses tanpa `index.php` (contoh: `https://simpmw.polsri.ac.id/dashboard`).
- Link yang dihasilkan oleh `base_url()` atau `site_url()` secara otomatis akan mengecualikan `index.php`.

> [!NOTE]
> Pastikan di server Apache produksi, modul `mod_rewrite` telah diaktifkan dan folder root aplikasi memiliki izin `AllowOverride All`.
