# 📜 Kisah: Integrasi Pemilihan Mentor di Tahap 4

**Waktu**: 17 April 2026, 03:39 PM
**Topik**: Digitalisasi Penunjukkan Mentor (Praktisi) pada Tahap Wawancara Perjanjian.

---

## 🚀 Apa yang Dilakukan?

Hari ini kita berhasil menghubungkan "Missing Link" antara fase seleksi dan fase pendampingan. Sebelumnya, mentor (praktisi) belum ditunjuk secara sistematis pada saat mahasiswa melakukan tanda tangan kontrak (Tahap 4). 

Sekarang, Admin memiliki otoritas penuh untuk menunjuk **1 Mentor untuk 1 Tim** langsung pada saat memvalidasi berkas perjanjian mereka.

### Perubahan Utama:
1.  **Database Upgrade**: Menambahkan kolom `mentor_id` pada tabel `pmw_proposals`. Ini memungkinkan kita melacak siapa mentor yang ditunjuk sejak awal (bahkan sebelum data dipindahkan ke tabel `pmw_teams` di fase akhir).
2.  **Smart Validation**: Pada halaman **Detail Perjanjian**, Admin kini melihat dropdown "Pilih Mentor Praktisi". Dropdown ini wajib diisi sebelum verifikasi disimpan, memastikan tidak ada tim yang lolos tanpa mentor.
3.  **Enhanced Visibility**: Pada daftar antrian perjanjian (Index), sekarang muncul kolom baru "Mentor (Praktisi)" untuk mempermudah monitoring distribusi beban kerja mentor.

## 🛠 Detail Teknis
- **Migration**: `2026-04-17-083848_AddMentorIdToProposals.php`
- **Model**: Update `PmwProposalModel` untuk mendukung `mentor_id` dan join query ke `pmw_mentors`.
- **Controller**: `Admin\WawancaraController` dimodifikasi untuk menangani pengambilan data mentor dan penyimpanan ID mentor terpilih.
- **UI/UX**: Penambahan komponen `select` dengan desain premium (icon `fa-user-tie`) dan penambahan kolom mentor di index.

## 📌 Status Saat Ini
- Pemilihan mentor sudah fungsional.
- Data `mentor_id` tersimpan dengan benar di tabel proposal.
- Siap digunakan untuk tahap berikutnya (Tahap 7: Mentoring) di mana mentor akan mulai melakukan absensi/log kegiatan mahasiswa.
