# Kisah Perbaikan Fitur Expo Mahasiswa (Part 2)

Log ini mencatat finalisasi perbaikan fitur Expo, terutama pada bagian akses berkas dan UI modal update.

## Masalah Utama
1. **Akses Ditolak**: Preview gambar/berkas menggunakan rute galeri kegiatan yang tidak memiliki validasi untuk data Expo.
2. **UI Update**: Modal update dokumentasi tidak menampilkan berkas yang sudah ada, sehingga membingungkan mahasiswa.
3. **Logika Update**: Backend hanya mendukung penambahan berkas baru, bukan pembaruan berkas/judul yang sudah ada.

## Perubahan yang Dilakukan

### 1. Controller (`Mahasiswa/ExpoController.php`)
- Menambahkan rute internal `viewAttachment` untuk melayani berkas dari `WRITEPATH` dengan validasi kepemilikan.
- Menambahkan rute `deleteAttachment` (DELETE) untuk mendukung penghapusan berkas via AJAX dari UI.
- Memastikan rute `viewCertificate` tetap terjaga.

### 2. Service (`PmwExpoService.php`)
- Memperbarui `submitDocumentation` untuk menangani array `attachment_ids[]`.
- Jika ID ada, sistem akan memperbarui judul atau mengganti berkas fisik jika diunggah berkas baru.
- Jika ID tidak ada, sistem akan menambah lampiran baru.

### 3. Rute (`app/Config/Routes.php`)
- Mendaftarkan rute GET `expo/attachment/(:num)` dan DELETE `expo/attachment/(:num)`.

### 4. View (`app/Views/mahasiswa/expo.php`)
- **Inisialisasi Alpine.js**: Variabel `attachments` sekarang diisi dari data PHP yang sudah ada.
- **Preview**: Mengarahkan URL preview ke rute expo yang baru (`mahasiswa/expo/attachment/`).
- **Modal UI**: 
    - Menampilkan status "Sudah Ada" untuk berkas lama.
    - Menambahkan tombol hapus yang terintegrasi dengan backend via AJAX.
    - Mendukung penggantian berkas pada slot yang sudah ada.

## Hasil
Mahasiswa sekarang dapat melihat preview berkas mereka tanpa error "Akses Ditolak", melihat berkas apa saja yang sudah diunggah di modal update, serta melakukan manajemen berkas (tambah/hapus/ganti) dengan lancar.
