# Kisah Perbaikan Bug Upload Dokumentasi Expo

## Masalah
Mahasiswa mengalami error `finfo_file(/tmp/...): Failed to open stream: No such file or directory` saat mencoba mengirimkan dokumentasi Expo.

## Analisis
Error ini terjadi di `PmwExpoService::submitDocumentation` karena pemanggilan `$file->getMimeType()` dilakukan **setelah** file dipindahkan menggunakan `$file->move()`. 
Di CodeIgniter 4, setelah file dipindahkan dari direktori temporary (`/tmp`), objek `UploadedFile` tidak lagi dapat mengakses file tersebut di lokasi asalnya untuk keperluan deteksi MIME type jika belum di-cache.

## Solusi
Memindahkan logika pengambilan MIME type ke variabel `$mimeType` **sebelum** perintah `$file->move()` dijalankan.

## Perubahan
- File: `app/Services/PmwExpoService.php`
- Menambahkan `$mimeType = $file->getMimeType()` sebelum `$file->move()`.
- Menggunakan variabel `$mimeType` untuk menentukan `file_type` ('image' atau 'document').

## Status
✅ Perbaikan telah diterapkan.
