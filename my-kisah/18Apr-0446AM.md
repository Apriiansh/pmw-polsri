# Kisah: Flexible Expenditure & Improved UI Feedback in Guidance Logbook

## 18 April 2026 - 04:46 AM

### Log Overview
Refined the Mahasiswa Guidance Logbook module to provide a more flexible expenditure tracking system (Nota Details) and improved user feedback for file uploads.

### Technical Changes
1. **Database Schema**:
   - Created migration `AddNotaDetailsToLogbooks` to add `nota_title`, `nota_qty`, and `nota_price` to the `pmw_guidance_logbooks` table.
   - Applied migration successfully via `php spark migrate`.

2. **Backend Logic**:
   - **Model**: Updated `PmwGuidanceLogbookModel` `$allowedFields` to include the new columns.
   - **Service**: Updated `PmwGuidanceService::submitLogbook()` to capture and persist the itemized nota data.

3. **Frontend & UX**:
   - **Structured Input**: Replaced the flat nominal field with a structured "Rincian Konsumsi" grid.
   - **Alpine.js Logic**:
     - Implemented `calculateTotal()` to automatically update the nominal amount based on quantity and price.
     - Implemented `handleAssignmentChange()` to provide real-time filename feedback for PDF uploads, resolving the "no feedback" issue.
   - **Visual Improvements**:
     - Added conditional banners for approved logbooks.
     - Improved photo preview logic to handle existing files vs. new selection more gracefully.
     - Standardized spacing and alignment using the established design system.

### State of Feature
- **Status**: ✅ Completed & Migrated.
- **Next Steps**: Dosen and Admin views might need minor updates to display these itemized details in their verification screens.
