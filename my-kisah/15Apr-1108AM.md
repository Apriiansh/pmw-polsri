# Kisah Progress - 15Apr 11:08AM

## Ringkasan
Sesi ini fokus pada perapihan UI admin dan auth, serta penambahan fitur manajemen status user dan detail user (modal), kemudian perbaikan alur registrasi agar NIM dan username tidak bentrok serta sesuai requirement Shield.

## Perubahan Utama yang Dilakukan

### 1) Admin Users List (`app/Views/admin/users.php`)
- Ditambahkan tombol **Toggle Status** untuk **Activate/Deactivate** user langsung dari list.
- Ditambahkan tombol **Detail** (ikon mata) yang membuka **modal** detail user.
- Modal menampilkan:
  - **Detail Akun** (User ID, Email, Role, Status, Terdaftar)
  - **Detail Role** (role-specific) yang dinamis sesuai profile:
    - Mahasiswa: NIM, Nama, Jurusan, Prodi, Semester, No HP, Gender
    - Dosen: NIDN, Nama, Jurusan, Prodi, No HP, Gender
    - Mentor: Nama, Perusahaan, No HP, Email, Jabatan, Keahlian
    - Reviewer: NIP/NIDN, Nama, Institusi, No HP, Email, Spesialisasi
- Layout modal diubah menjadi **2 kolom (menyamping)**: "Detail Akun" dan "Detail (Role)".

### 2) Admin Controller (`app/Controllers/AdminController.php`)
- Di method `users()` ditambahkan load data:
  - groups per user
  - profile role-specific per user (mahasiswa/dosen/mentor/reviewer)
  sehingga data siap dipakai untuk modal detail.

### 3) Routing (`app/Config/Routes.php`)
- Sudah ada route untuk toggle status user: `admin/users/toggle-status/(:num)`.

### 4) Auth - Register Flow

#### a) Register View (`app/Views/auth/register.php`)
- Field registrasi diganti menjadi **NIM** (`name="nim"`).
- Ditambahkan hidden field **`username`** yang dikirim saat submit:
  - `name="username"` di-generate via Alpine state.
- Ditambahkan generator username di frontend dengan pola **B**:
  - Ambil **nama tengah** (kata ke-2), fallback ke **nama terakhir**, lalu tambahkan **4 digit terakhir NIM**.

#### b) Register Controller (`app/Controllers/AuthController.php`)
- Validasi disesuaikan: `nim` dicek unique di `pmw_profiles.nim`.
- `username` tetap dipastikan unik di `users.username` (di-generate server-side juga).
- Username server-side mengikuti pola yang sama (nama tengah fallback nama terakhir + last4 NIM).
- Profile mahasiswa (`pmw_profiles`) tetap dibuat dengan NIM lengkap (`nim`).

### 5) Toast Position di Auth Pages (`app/Views/layouts/public.php`)
- Toast notification dipindah agar **di halaman auth** tampil **top-center (di atas card)**.
- Di halaman non-auth tetap tampil **top-right** seperti sebelumnya.

## Catatan Teknis
- Penyebab error awal "The Username field is required." adalah karena Shield mengharuskan `users.username` terisi saat pembuatan user, sementara form register sempat hanya mengirim `nim` tanpa `username`. Ini diperbaiki dengan hidden field + generator frontend dan generator backend sebagai source-of-truth.
