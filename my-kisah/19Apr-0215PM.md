# Kisah 19 April - Fix Pitching Desk Auto-Download

## Masalah
Saat mengakses halaman detail validasi Pitching Desk (Dosen/Admin), browser secara otomatis mendownload file PPT/PDF. Hal ini disebabkan oleh respons server dari link dokumen yang dianggap sebagai request download oleh browser, padahal seharusnya hanya ditampilkan di dalam iframe (untuk PDF).

## Solusi
1.  Memperbarui logika `viewDoc` di semua controller Pitching Desk (**Dosen**, **Mahasiswa**, **Admin**).
2.  Menambahkan deteksi parameter `?inline=1`. Jika ada, server akan mengirimkan header `Content-Disposition: inline` secara eksplisit dan menggunakan `setBody` untuk menyajikan konten file tanpa memicu download otomatis.
3.  Memperbaiki deteksi ekstensi file di view `validation_detail.php` agar lebih tangguh dengan menggunakan `original_name` dan `file_path` sebagai fallback.
4.  Melebarkan ukuran modal detail mahasiswa di `shared/_monitoring_team.php` dari `max-w-md` menjadi `max-w-xl` untuk menghindari tampilan yang sempit.
5.  Konsolidasi header monitoring di `dosen/monitoring` dan `mentor/monitoring` untuk menghilangkan redundansi informasi dan menyatukan navigasi (tombol Back dan Jadwalkan).

## Hasil
- Akses ke halaman detail tidak lagi memicu download otomatis.
- PDF tetap dapat dilihat secara langsung di dalam iframe (preview).
- Tombol "Download" tetap berfungsi sebagaimana mestinya untuk mengunduh file asli.
- Tampilan detail mahasiswa lebih lega dan profesional.
- UI Monitoring lebih bersih dengan satu header terpadu yang memuat semua informasi penting dan aksi cepat.
