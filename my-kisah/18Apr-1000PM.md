# Kisah: Perbaikan Akses & Logic Logbook Kegiatan

**Waktu:** 18 April 2026, 10:00 PM
**Status:** Perbaikan Bug & Hardening UI

## Apa yang terjadi?
Ditemukan beberapa error kritikal pada modul Logbook Kegiatan (Tahap 9) setelah migrasi ke CI4 Entity:
1. **Error Scope Alpine.js:** Fungsi `handleMouseMove` tidak terbaca di dalam loop Bento cards karena masalah scoping.
2. **Entity vs Array Error:** View Dosen dan Mentor mencoba mengakses data Entity menggunakan sintaks array (`$logbook['...']`), yang menyebabkan runtime error.
3. **Invalid File Requests:** Mahasiswa view mencoba melakukan request foto supervisor meskipun logbook belum ada (ID 0), menyebabkan error "Berkas tidak ditemukan".

## Solusi yang diimplementasikan
1. **Global Alpine Scope:** Memindahkan logika `handleMouseMove` ke root container `x-data` di file `mahasiswa/activity.php`.
2. **Refactor Entity Access:** Mengubah semua akses data logbook di `dosen/activity/index.php` dan `mentor/activity/index.php` menjadi format objek (`$logbook->property`).
3. **UI Hardening:** Menambahkan pengecekan yang lebih ketat pada rendering gambar di sisi Mahasiswa. Gambar hanya akan direquest jika `$logbook` valid dan memiliki ID. Penggunaan ternary PHP dipastikan aman dari warning null property.

## Hasil Akhir
- Modul Mahasiswa kini stabil saat berpindah antar tab draft/submit.
- Modul Dosen dan Mentor dapat membuka modal verifikasi dengan lancar.
- Konsol browser bersih dari error `undefined` maupun 404 pada request file awal.

**Langkah Selanjutnya:**
- Melakukan pengetesan akhir pada alur approval berjenjang (Mhs -> Dosen -> Mentor -> Admin).
