# Catatan Perbaikan - 15 April 2026 (01:37 AM)

## 1. Perbaikan Layout & Navbar
- **Flickering & Positioning**: Memperbaiki masalah garis hitam (flickering) dan memposisikan ulang top bar agar lebih stabil.
- **User Info**: Mengubah tampilan di top bar untuk menampilkan nama lengkap user (mengambil dari tabel profile). Jika nama kosong, baru menampilkan username dari data autentikasi.
- **Sidebar**: Meningkatkan tampilan sidebar saat dikecilkan (collapsed) agar ikon tetap terlihat jelas dan proporsional.

## 2. Perbaikan Manajemen User (Form & Controller)
- **Fungsionalitas Role**: Memperbaiki radio button pemilihan role yang sebelumnya tidak responsif.
- **Validasi Frontend (Alpine.js)**:
  - Melakukan refaktorisasi inisialisasi script Alpine.js menjadi fungsi `userForm()` untuk kemudahan pengelolaan.
  - Menerapkan atribut `:disabled` dan `:required` secara dinamis pada field spesifik peran (NIM untuk Mahasiswa, NIP untuk Dosen, Company untuk Mentor, dsb).
  - Ini mencegah browser memblokir pengiriman form karena field tersembunyi yang memiliki atribut `required`.
- **Integritas Data Backend**:
  - Menambahkan sistem **Database Transactions** (`transStart`/`transComplete`) pada `AdminController::storeUser` dan `AdminController::updateUser`.
  - Ini memastikan data akun user dan data profil dibuat secara atomik; jika salah satu gagal, maka perubahan akan dibatalkan (mencegah data yatim-piatu).
  - Menambahkan aturan validasi yang dinamis berdasarkan role yang dipilih untuk memastikan semua field wajib terisi dengan benar.
- **Profil Admin**: Menambahkan logika pembuatan record profil dasar untuk peran Admin, sehingga nama mereka dapat tersimpan dan tampil di layout utama.

## 3. Optimasi Penanganan Error
- Memastikan pesan error dari backend tertampil dengan jelas di form jika terjadi kesalahan validasi baik pada level user maupun level profil.
